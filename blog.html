May 26th, 2021

It's day 3 of me learning html, and I am genuinely excited to now know how to add pictures (excitement specifically for GIFs) and tables to documents!

For context, this is what I have learnt so far:

  • The history and evolution of html
    • Day 1 was more of a history than hands-on coding lesson.
  • How to create a file (I know, impressive.)
  • The semantics of html
  • Stylistic text elements
  • Structurally-based elements
  • Hyperlinks
    • Absolute v. relative paths
    • Linking to email addresses
    • Opening links to a new window
    • Linking to specific parts of the same page
  • How to create and nest lists (obviously...)
    • Ordered lists
      • Start attribute
      • Reversed attribute
    • Unordered lists
    • Descriptive lists

Well, that's a lot; I definitely missed some things too. It's funny how easy it is to cognitively minimize large quantities. Then you write them down and are forced to accept their real value. 🤷

I've been debating whether or not to jump into CSS (because I know that's where a lot of the style comes into play), but I think it's best to become fluent with html first.

May 28th, 2021

Today I continued on with the Coursera videos, which I now find a bit redundant since I did all of the assigned Shay Howe readings beforehand. One thing the professor does stress that is not included in the readings is the importance of accessibility when coding. She frequently references how some people use screenreaders and how critical it is to keep them in mind. For example, screenreaders cannot always read images or icons, which is why one should be careful when creating graphic hyperlinks and be sure to include good alt text.

The professor's emphasis on accessibility very much resonates with me, for it reflects my family science (aka public health) background and value of empathy. The more I learn about and practice html, the more I appreciate its linguistic nature. While the semantics were annoying at first, they're growing on me. Html really is a beautiful thing.

I also learnt how to source from Font Awesome and use multimedia tags today (i.e. how to add audio and video to pages). In honor of the latter, enjoy the following song (that has been my go-to replay as of late):


Kenny, by Still Woozy

May 29th

Tonight I learned some (what the Coursera professor called) useful tags, a few of which require JavaScript to work, but nevertheless useful.

Some of my favorites include:

  • <details> with <summary>
  • <iframe>
  • <progress>
  • <button>
  • <code> (obviously)

<progress> and <button> both require JavaScript to work, which marks a strengthened commitment of mine to learnt both CSS and JavaScript. As an artist (and like I've mentioned before), html is painfully bland, and I know CSS is what will enable me to create the aesthetic I envision by cannot yet manifest. JavaScript seems to be important as a metadata-ish device, something that is not technically seen but influences the functionality of seen elements. (Disclaimer: I know absolutely nothing about JavaScript and internally feel that assessment being incorrect, but this is a reflection, isn't it?)

At this point, I feel that I pretty much know html. Am I an expert? Of course not. But even by day 3, when I knew how to create webpages and basic semantic and stylistic elements, I was pretty much set. It seems like html is the kind of language you continuously learn via practice and researching tags when necessary. The Coursera professor herself said it was impossible for her to teach us all the tags, and in the future, you just look up the ones you need but not yet know. Just like any other language, I'll continue learning new “words,” “grammar,” and “literary devices” along the way.

What a productive week it has been.
